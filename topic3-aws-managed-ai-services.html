📝 Text & Language Services
🧠 NLP
Amazon Comprehend
  • Fully managed, serverless NLP service
  • Detects: language, key phrases, entities, sentiment, PII
  • Tokenization + parts of speech analysis
  • Auto-organizes text into topics
Custom Models:
  • Custom Classification: Organize docs into YOUR categories. Supports text, PDF, Word, images. Real-time (single doc) or Async (batch).
  • Custom Entity Recognition (NER): Extract specific terms (policy numbers, escalation phrases). Train with your own data. Real-time or Async.
  • Model versioning managed by Comprehend. Can copy models cross-account (same region, need ARN + KMS key).
⚡ Exam: Comprehend = NLP insights. Custom NER = business-specific entity extraction. Custom Classification = your own document categories.
🌐 Translation & Speech-to-Text
Amazon Translate
  • Natural language translation at scale
  • Localize websites, apps for international users
  • Translate large volumes of text efficiently
Amazon Transcribe
  • Speech → Text via ASR (Automatic Speech Recognition)
  • Auto-removes PII via Redaction
  • Automatic Language Identification (multi-lingual audio)
  • Toxicity Detection: 7 categories: sexual harassment, hate speech, threat, abuse, profanity, insult, graphic
Improving Transcribe Accuracy:
ToolBest For
Custom VocabulariesSpecific words, brand names, acronyms, pronunciation hints
Custom Language ModelsDomain-specific context (large volumes of domain speech)
Both togetherHighest accuracy ⭐
Use cases: Customer service transcription, closed captioning, searchable media archives.
🔊 Text-to-Speech & OCR
Amazon Polly
  • Turn text into lifelike speech using deep learning
  • Lexicons: Define how to read specific text (AWS → "Amazon Web Services")
  • SSML: Markup to control pronunciation ("Hello, <break> how are you?")
  • Voice engines: generative, long-form, neural, standard
  • Speech marks: Encode word/sentence start+end positions for lip-sync or word highlighting
Amazon Textract
  • Extracts text, handwriting, data from scanned documents
  • Extracts from forms AND tables
  • Supports: PDFs, images, and more
Use Cases:
Financial invoicesMedical recordsTax formsID documentsPassports
⚡ Exam: Polly = text → speech. Textract = document → structured data (forms + tables). Polly SSML = fine-grained speech control.
👁 Vision, Search & Conversation
📸 Computer Vision
Amazon Rekognition
Capabilities:
Object/Scene Detection Facial Analysis Face Search & Verify Celebrity Recognition Text Detection Content Moderation Pathing (sports)
Custom Labels:
  • Train Rekognition on YOUR images (only needs a few hundred)
  • Find your own logos, products, custom objects
  • Upload labeled images → creates custom model
Content Moderation:
  • Detect inappropriate/offensive content in images & video
  • Reduces human review to 1–5% of content volume
  • Integrated with Amazon A2I for human review of low-confidence results
  • Custom Moderation Adaptors: Provide your own labeled images to extend/enhance moderation
  • API: DetectModerationLabels
⚡ Exam: Rekognition + A2I = human-in-the-loop for low-confidence moderation. Custom Labels needs only hundreds of images.
💬 Chatbots & Search
Amazon Lex
  • Build chatbots with voice + text using deep learning
  • Supports multiple languages
  • Understands user intent → invokes Lambda to "fulfill the intent"
  • Asks for Slots (input parameters) when needed
  • Integrates with: Lambda, Connect, Comprehend, Kendra
Amazon Kendra
  • Fully managed document search powered by ML
  • Natural language search — answers from within documents
  • Supports: text, PDF, HTML, PowerPoint, MS Word, FAQs
  • Incremental Learning: Learns from user feedback to promote preferred results
  • Manual fine-tuning: data importance, freshness, custom
Kendra Data Sources:
S3RDSGoogle DriveSharePointOneDrive3rd party / Custom
⚡ Lex vs Kendra: Lex = chatbot conversations. Kendra = enterprise document search ("Where is the IT desk?" → "1st floor").
👤 Human-in-the-Loop
Amazon Augmented AI (A2I)
  • Human oversight of ML predictions in production
  • High-confidence predictions → returned to app immediately
  • Low-confidence predictions → sent for human review
  • Reviews consolidated with weighted scores → stored in S3
  • Reviewed data can be added back to training dataset
Workforce Options:
  • Your own employees
  • 500,000+ contractors from AWS
  • Mechanical Turk (some pre-screened for confidentiality)
Works with:
RekognitionTextractCustom ML modelsSageMaker
⚡ Exam: A2I = human review for low-confidence ML predictions. Integrates tightly with Rekognition (content mod) and Textract (document extraction).

🔍 Anomaly Detection
Amazon Lookout
  • ML-powered anomaly detection — detects unusual variances
  • Measure = the metric being optimized
  • Dimensions = features that influence the measure
Three Flavors:
ServiceUse Case
Lookout for Metrics Business KPIs: revenue, ad campaigns, user metrics. Inputs: S3, Redshift, Athena, CloudWatch, Salesforce, Marketo. Outputs: SNS, Lambda.
Lookout for Equipment Industrial sensors → predictive maintenance. Detect equipment failures early.
Lookout for Vision Visual quality inspection via computer vision (manufacturing defects).
⚡ Exam: Lookout for Metrics = business anomalies. Lookout for Equipment = IoT/industrial sensors. Lookout for Vision = visual quality inspection.
🚨 Fraud Detection
Amazon Fraud Detector
  • Fully managed ML for fraud detection
  • Automatic model creation, continuous learning
  • Feature importance insights (model variables)
  • Ingests data from S3 or API
  • Supports rule-based actions
  • SageMaker integration
Fraud Examples:
Online paymentsNew accountsTrial abuseAccount takeovers
Workflow:
  • Choose use case → create event type + entities
  • Define data source → define labels
  • Create + train model → review performance → deploy
⚡ Exam: Fraud Detector = automatic ML for fraud, no ML expertise needed. Handles ETL + feature engineering + model selection + training + deployment internally.

⚠️ Exam Traps & Key Distinctions
🎯 Must-Know Distinctions
  • Comprehend vs Transcribe: Comprehend = analyze existing text. Transcribe = audio → text.
  • Lex vs Kendra: Lex = chatbot conversations with intents. Kendra = search documents with NL questions.
  • Rekognition vs Textract: Rekognition = images/video analysis. Textract = structured data from document scans.
  • Q Business vs Q Developer: Q Business = employees + company data. Q Developer = developers + AWS/coding.
  • Personalize Recipes: Always for recommendations. Different recipe = different recommendation strategy.
  • Lookout flavors: Metrics = business KPIs. Equipment = industrial IoT. Vision = visual quality inspection.
  • A2I workflow: High confidence → auto-approved. Low confidence → human review → improve training data.
  • Custom Transcribe: Custom Vocabulary = specific words. Custom Language Model = domain context. Use BOTH for highest accuracy.
⚠️ Common Exam Traps
  • Q Business is built on Bedrock but you CANNOT choose the underlying FM
  • Q Business requires IAM Identity Center — not just IAM users
  • Personalize Recipes = algorithm templates — you still need to provide training config
  • Rekognition Custom Labels needs only a few hundred images (not thousands)
  • A2I vendors can be pre-screened for confidentiality requirements
  • Polly SSML uses XML tags — e.g., <break> to pause
  • Transcribe Toxicity Detection uses both speech cues (tone/pitch) AND text
  • Comprehend Custom Models can be copied cross-account (same region, need ARN)
  • Fraud Detector ingests from S3 or an API — not Kinesis directly
  • Trainium = training chips. Inferentia = inference chips. Don't confuse.
