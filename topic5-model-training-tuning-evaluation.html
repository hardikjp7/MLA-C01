🧠 Deep Learning Fundamentals
⚡ Neural Network Types
Feedforward (MLP)
Standard layers, no loops. Classification, regression.
CNN (Convolutional)
Images, feature-location invariant. Local receptive fields + convolutions + pooling. Architectures: LeNet-5AlexNetGoogLeNetResNet
RNN (Recurrent)
Sequential data, time series. Feedback loop = "memory cell". Trained via Backpropagation Through Time (BPTT).
LSTM (short + long-term state)
GRU (simplified LSTM, similar performance)
Transformer
Self-attention replaces RNNs. Parallelizable. BERT, GPT, T5, DistilBERT.
⚡ CNN topology: Conv2D → MaxPooling2D → Dropout → Flatten → Dense → Dropout → Softmax
⚡ Activation Functions
Linear
Does nothing useful. No backprop.
Binary Step
On/off. No multi-class. Bad slopes.
Sigmoid
0→1. Vanishing gradient problem. Expensive.
TanH
-1→1. Preferred over Sigmoid. Good for RNNs.
ReLU ⭐
Most popular. Fast. "Dying ReLU" when input ≤ 0.
Leaky ReLU
Fixes dying ReLU. Negative slope below 0.
PReLU
ReLU with learned slope. Complex, YMMV.
Swish
Google's. Great for 40+ layer networks.
Softmax
Final layer of multi-class. Probabilities. Can't produce multiple labels (sigmoid can).
ELU / Maxout
Maxout = max of inputs. ReLU is special case. Doubles parameters.
⚡ Choosing: Multi-class output → Softmax. RNNs → TanH. Everything else → start with ReLU → Leaky ReLU → PReLU/Maxout (last resort) → Swish (deep nets)

⭐ Critical Recap (Exam Favorite!)
  • Small batch → less likely to get stuck in local minima
  • Large batch → may converge on wrong solution
  • Large learning rate → can overshoot correct solution
  • Small learning rate → increases training time
Regularization Techniques (Prevent Overfitting)
  • Dropout: Randomly zero out neurons during training
  • Early Stopping: Stop when validation loss stops improving
  • L1: Sum of weights → feature selection → sparse output
  • L2: Sum of squared weights → all features kept → dense output
L1 vs L2: Use L1 when you want feature selection (reduces some to 0). Use L2 when all features matter. L2 is computationally efficient.
💀 Gradient Problems & Fixes
Vanishing Gradient
Slope → 0. Deep nets + RNNs. Training slows or stops. Numerical errors.
Exploding Gradient
Opposite problem. Weights grow too large. Training diverges.
Fixes for Vanishing Gradient:
  • Multi-level hierarchy: Sub-networks trained individually
  • LSTM cells: Separate short + long-term memory
  • ResNet: Skip connections maintain gradient flow
  • Ensemble of shorter networks
  • Better activation function: Use ReLU (not sigmoid)

Transformers & BERT:
  • Self-attention: processes entire sequence in parallel (not word by word)
  • Q, K, V matrices → scaled dot-product attention → updated embeddings
  • GPT: Decoder-only, generates next token repeatedly. Masked self-attention.
  • BERT: Encoder-only. Bidirectional. Masked language modeling.
  • T5: Encoder + Decoder. Text-to-text.
  • DistilBERT: Knowledge distillation → 40% smaller than BERT

📈 ROC/AUC, P-R Curve & Regression Metrics
ROC Curve
  • Plot of True Positive Rate (Recall) vs False Positive Rate
  • At various threshold settings
  • Ideal = upper-left corner (TPR=1, FPR=0)
  • Points above diagonal = better than random
  • More bent toward upper-left = better model
AUC (Area Under ROC Curve)
  • Probability that classifier ranks a random positive instance higher than a random negative one
  • AUC = 0.5 Useless (random)   AUC = 1.0 Perfect
  • Most common metric for comparing classifiers
P-R Curve (Precision-Recall)
  • Better than ROC for information retrieval problems
  • When searching large number of docs for tiny relevant subset
  • ROC can show misleadingly small values in this case
Regression Metrics
MetricDescriptionUse
RMSERoot Mean Squared Error. Penalizes large errors more.Continuous value prediction
MAEMean Absolute Error. Average of absolute differences.Continuous, robust to outliers
R² (R-Squared)Square of correlation between observed and predicted. 0-1, higher = better.How well model fits data
⚡ Quick rule: Classification → Precision/Recall/F1/AUC. Regression → RMSE/MAE/R². Imbalanced data → Recall (if FN costly) or Precision (if FP costly).

🎯 SageMaker Automatic Model Tuning (AMT)
🔧 AMT Overview & Best Practices
  • Define: hyperparameters + ranges + objective metric
  • SageMaker spins up tuning jobs, learns as it goes
  • Best results → can be deployed as a model
Best Practices:
  • Don't optimize too many hyperparameters at once
  • Limit ranges to as small as possible
  • Use logarithmic scales when appropriate
  • Don't run too many jobs concurrently (limits learning)
  • Multi-instance jobs must report correct objective metric
⚡ Exam: AMT = Hyperparameter Tuning Job. Learns from previous runs. Concurrent runs reduce its ability to learn. Use logarithmic scales for learning rate etc.
🔍 Tuning Search Strategies
Grid Search
Limited to categorical parameters. Brute force — tries EVERY combination. Very slow.
Random Search
Random HP combination each job. No dependence on prior runs → can all run in parallel. Better than grid for continuous params.
Bayesian Optimization ⭐
Treats tuning as a regression problem. Learns from each run to converge faster. Sequential by nature.
Hyperband ⭐
Best for algorithms that publish iterative results (neural nets with epochs). Dynamic resource allocation. Early stopping. Parallel. Much faster than random or Bayesian.
⚡ Exam: Random search = fully parallel. Bayesian = sequential but smart. Hyperband = fastest for iterative training (epochs). Grid = categorical only, impractical.
🔄 AMT Advanced Features
Early Stopping:
  • Stops training job early if not improving objective
  • Reduces compute cost + avoids overfitting
  • Set "early stopping": "Auto"
  • Requires algorithm to emit objective metric after each epoch
Warm Start:
  • Uses previous tuning jobs as starting point
  • Two types:
  • IDENTICAL_DATA_AND_ALGORITHM — same dataset/algo, continue search
  • TRANSFER_LEARNING — different dataset/algo, transfer knowledge
Resource Limits:
  • Default limits: parallel tuning jobs, # hyperparameters, training jobs per tuning job
  • Increasing requires quota increase from AWS support
⚡ Warm Start: IDENTICAL = continue same tuning. TRANSFER_LEARNING = start from previous knowledge with different setup.

⚠️ Exam Quick-Fire
🎯 Critical Facts
  • Small batch → escapes local minima. Large batch → may converge wrong.
  • Large learning rate → overshoots. Small → slow convergence.
  • L1 → feature selection (some weights → 0). L2 → all features weighted.
  • Softmax → multi-class final layer. Sigmoid → multi-label.
  • ReLU → default choice. TanH → best for RNNs.
  • Recall = TP/(TP+FN). Use when FN is costly (fraud, disease).
  • Precision = TP/(TP+FP). Use when FP is costly (drug tests).
  • F1 = harmonic mean of Precision + Recall.
  • AUC 0.5 = useless. AUC 1.0 = perfect.
  • Hyperband = fastest tuning for iterative algorithms (epochs).
  • Warm Start IDENTICAL vs TRANSFER_LEARNING — key distinction.
  • Managed Spot Training = 90% savings + need checkpointing!
  • Model Parallelism (MPP) = PyTorch only. For LLMs >1B params.
  • AMT concurrent jobs ↑ → learning efficiency ↓
⚠️ Common Exam Traps
  • Accuracy is misleading on imbalanced data — use Precision/Recall/F1
  • ROC AUC vs P-R Curve: P-R better for information retrieval (many negatives)
  • Bagging = parallel, Boosting = sequential — don't confuse
  • Autopilot HPO if >100MB, Ensembling if <100MB (Auto mode)
  • Autopilot defaults to HPO when it can't read dataset size
  • Debugger is NOT compatible with SageMaker Distributed Training Libraries
  • Early Stopping in AMT requires algorithm to emit metric after each epoch
  • Multi-GPU does NOT help Linear Learner
  • Managed Spot Training needs checkpointing — or lose progress on interruption
  • FSx for Lustre = Single AZ (even though it's fast). Requires VPC setup.
  • LSTM = short + long-term states. GRU = simplified LSTM, similar performance.
  • DistilBERT = 40% smaller via knowledge distillation, not compression
