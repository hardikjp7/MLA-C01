📊 Supervised — Regression & Classification
Linear Learner
What it does
  • Regression (numeric) or Classification (binary/multi-class)
  • Fits a line; uses linear threshold for classification
Input Format
RecordIO-Protobuf (Float32) CSV
File or Pipe mode. First CSV column = label.
How it works
  • Auto-normalizes training data
  • Stochastic gradient descent (Adam, AdaGrad, SGD)
  • Multiple models optimized in parallel → best selected
  • L1 / L2 regularization tuning
Key Hyperparameters
learning_ratemini_batch_sizeL1wd (L2)balance_multiclass_weightstarget_precisiontarget_recall
Instances
CPU or GPU (single/multi-machine). Multi-GPU does NOT help.
⚡ Exam: Linear Learner = regression + classification in one algo. target_precision/target_recall = hold one metric while maximizing the other.
XGBoost
What it does
  • Boosted decision trees — each new tree corrects errors of previous
  • Classification + Regression. Gradient descent minimizes loss.
  • Kaggle competition winner — very popular
Input Format
CSVlibsvmrecordIO-protobufParquet
Serialized with Pickle. Use as framework (sagemaker.xgboost) or built-in algo.
Key Hyperparameters
subsampleprevents overfitting
etastep size shrinkage → prevent overfitting
gammamin loss reduction for partition; larger = conservative
alpha/lambdaL1/L2 regularization; larger = more conservative
scale_pos_weightimbalanced classes: sum(neg)/sum(pos)
max_depthtree depth; too high → overfit
eval_metricAUC, error, rmse
Instances
Memory-bound → M5. GPU: P2/P3 (set tree_method=gpu_hist). v1.5+: distributed GPU (set use_dask_gpu_training=true).
⚡ Exam: XGBoost = memory-bound (M5 preferred). scale_pos_weight = fix imbalanced classes. alpha=L1, lambda=L2, gamma=conservative splits.
LightGBM
What it does
  • Gradient Boosting Decision Tree (like XGBoost/CatBoost)
  • Extended with GOSS + EFB (Gradient-based One-Side Sampling + Exclusive Feature Bundling)
  • Classification, Regression, or Ranking
Input Format
txt / csv
Training + optional validation channels.
Key Hyperparameters
learning_ratenum_leavesfeature_fractionbagging_fractionbagging_freqmax_depthmin_data_in_leaf
Instances
Single or multi CPU. Memory-bound → M5 not C5. Define instances with instance_count.
⚡ LightGBM vs XGBoost: LightGBM also does Ranking (XGBoost doesn't natively). Both memory-bound → M5. LightGBM input is txt/csv only.
📝 NLP & Sequence Algorithms
Seq2Seq
What it does
  • Input sequence of tokens → Output sequence of tokens
  • Machine Translation, Text Summarization, Speech-to-text
  • Implemented with RNN + CNN + Attention
Input Format
RecordIO-Protobuf
Tokens must be INTEGERS (unusual!). Must provide training + validation + vocabulary files.
Optimization Metrics
AccuracyBLEU scorePerplexity
BLEU = compares against multiple reference translations.
Instances
GPU only (P3). Single machine only (but multi-GPU on that machine OK). Training can take days.
⚡ Exam: Seq2Seq = GPU only, single machine. Tokens must be integers (not float). Three required data channels: train + validation + vocabulary.
BlazingText
Two Modes
Text Classification (Supervised)
Predict labels for sentences. Web search, info retrieval. Input: __label__X sentence… or augmented manifest format.
Word2Vec (Unsupervised)
Creates word embeddings. Semantically similar words = vectors close together. Words only, not sentences/docs. Used in NLP: translation, sentiment.
cbowskipgrambatch_skipgram
Key Hyperparameters
modelearning_ratewindow_sizevector_dimword_ngramsepochs
Instances
cbow/skipgram: ml.p3.2xlarge. batch_skipgram: multi-CPU. Text classification <2GB: C5; larger: P2/P3.
⚡ Exam: BlazingText = text classification + Word2Vec. Word2Vec = words only (not sentences). batch_skipgram = multi-CPU (distributed).
Object2Vec
What it does
  • Word2Vec generalized to arbitrary objects
  • Creates dense embeddings of high-dimensional objects
  • Nearest neighbors, cluster visualization, recommendations, genre prediction
Input Format
JSON Lines
Tokenized integer pairs: user-item, sentence-sentence, customer-customer, product-product.
Architecture
  • Two input channels → two encoders → comparator → FFNN
  • Encoder choices: avg-pooled embeddings, CNNs, Bidirectional LSTM
Key Hyperparameters
enc1_networkenc2_networkdropoutepochslearning_rate
Encoder options: hcnn, bilstm, pooled_embedding
Instances
Single machine only (multi-GPU OK). ml.m5.2xlarge, ml.p2.xlarge. Inference: ml.p3.2xlarge. Set INFERENCE_PREFERRED_MODE for encoder embeddings.
⚡ Exam: Object2Vec = Word2Vec for ANY objects. Single machine only. Two encoders + comparator architecture.
⏱ Time Series & Forecasting
DeepAR
What it does
  • Forecasting 1D time series data using RNNs
  • Train ONE model on many related time series simultaneously
  • Automatically finds frequencies and seasonality
Input Format
JSON LinesGzipParquet
Required: start (timestamp), target (values)
Optional: dynamic_feat (promotions, etc.), cat (categorical)
Best Practices
  • Always include entire time series for train/test/inference
  • Train on many time series, not just one
  • Don't use prediction length > 400
  • context_length can be smaller than seasonality (model lags 1 year)
Key Hyperparameters
context_lengthepochsmini_batch_sizelearning_ratenum_cells
Instances
CPU or GPU, single or multi. Start CPU (ml.c4.2xlarge). GPU helps only for larger models or batch >512. CPU-only for inference.
⚡ Exam: DeepAR = time series forecasting with RNNs. Key strength = one model for MANY time series. Always use full time series. CPU for inference.
Random Cut Forest
What it does
  • Anomaly detection — Unsupervised
  • Detects spikes, breaks in periodicity, unclassifiable points
  • Assigns an anomaly score to each data point
  • Amazon's own algorithm (they're very proud of it 😄)
Input Format
RecordIO-ProtobufCSV
File or Pipe mode. Optional test channel for accuracy/precision/recall/F1.
How it works
  • Forest of trees = partitions of training data
  • Measures expected complexity change when adding a point
  • Also works on streaming data (Kinesis Analytics)
Key Hyperparameters
num_treesnum_samples_per_tree
num_trees↑ = less noise. num_samples_per_tree: 1/value ≈ ratio of anomalous to normal data.
Instances
Does NOT benefit from GPU. Training: M4, C4, C5. Inference: ml.c5.xl.
⚡ Exam: RCF = anomaly detection (unsupervised). No GPU benefit. Also in QuickSight + Kinesis Analytics (streaming). num_samples_per_tree set based on anomaly ratio.
🖼 Computer Vision Algorithms
Image Classification
What it does
  • Assigns one or more labels to whole image
  • Doesn't locate objects — just classifies what's IN the image
Variants & Modes
MXNetFull training (random weights) OR Transfer learning (pretrained + random top layer). Default: 3ch 224×224.
TensorFlowTF Hub models: MobileNet, Inception, ResNet, EfficientNet. Top layer fine-tunable.
Input Format
RecordIOImage format (jpg/png)Augmented Manifest
Key Hyperparameters
batch_sizelearning_rateoptimizerweight_decay
Instances
GPU for training (p2, p3, g4dn, g5). Multi-GPU + multi-machine OK. CPU or GPU for inference.
Object Detection
What it does
  • Identifies ALL objects in image with bounding boxes
  • Outputs: object categories + confidence scores
  • Train from scratch OR use pretrained (ImageNet)
Variants
MXNetCNN + SSD (Single Shot multibox Detector). Backbone: VGG-16 or ResNet-50. Internal augmentation (flip, rescale, jitter).
TensorFlowResNet, EfficientNet, MobileNet from TF Model Garden.
Input Format
RecordIOjpg/png + JSON annotations
Key Hyperparameters
mini_batch_sizelearning_rateoptimizer (sgd, adam, rmsprop, adadelta)
Instances
GPU for training. Multi-GPU + multi-machine OK. CPU or GPU for inference.
Semantic Segmentation
What it does
  • Pixel-level object classification
  • Produces a segmentation mask
  • Use cases: self-driving, medical imaging, robot sensing
≠ Image Classification (whole image labels)
≠ Object Detection (bounding boxes)
✅ Pixel-by-pixel classification
Input Format
JPG imagesPNG annotations
Label maps for annotations. Augmented manifest for Pipe mode.
Algorithms & Backbones
Built on MXNet Gluon CV.
FCNPSPDeepLabV3
ResNet50ResNet101
Key Hyperparameters
algorithmbackboneepochslearning_rate
Instances
GPU for training. Multi-GPU + multi-machine OK.
⚡ Vision hierarchy: Image Classification (what's in image) → Object Detection (where is it, with box) → Semantic Segmentation (exact pixel outline)
📂 Topic Modeling, Clustering & Dimensionality
NTM
Neural Topic Model
  • Organize docs into topics (unsupervised)
  • Algorithm: Neural Variational Inference
  • You define how many topics
Input
recordIO-protobufCSV
Words tokenized to integers. 4 channels: train (required), validation, test, auxiliary (vocabulary).
Key HPs
num_topicsmini_batch_sizelearning_rate
Instances
GPU for training. CPU for inference (cheaper).
LDA
Latent Dirichlet Allocation
  • Topic modeling — not deep learning
  • Unsupervised; topics are unlabeled groupings
  • Also for: customer clustering by purchases, music harmonic analysis
Input
recordIO-protobufCSV
Pipe mode: recordIO only.
Key HPs
num_topicsalpha0
alpha0 small = sparse topics. alpha0 >1.0 = uniform topics.
Instances
Single-instance CPU only.
LDA vs NTM: Both topic modeling. LDA = CPU-based, cheaper. NTM = deep learning, GPU. Try both!
K-Means
Unsupervised Clustering
  • Divide into K groups by Euclidean distance
  • "Extra" centers → reduced to k (k=k*x). Uses k-means++ init.
  • Lloyd's method for final reduction
Input
recordIO-protobufCSV
Train: ShardedByS3Key. Test: FullyReplicated.
Key HPs
K!mini_batch_sizeextra_center_factorinit_method
Choose K: use Elbow Method (plot within-cluster sum of squares).
Instances
CPU recommended. GPU: only 1 GPU per instance used → ml.g4dn.xlarge if GPU.
PCA
Principal Component Analysis
  • Dimensionality reduction (unsupervised)
  • Projects to fewer dimensions preserving max variance
  • First component = largest variability
Input
recordIO-protobufCSV
Modes
Uses covariance matrix + SVD decomposition.
Regular
sparse data, moderate obs/features
Randomized
large obs + features, approx algorithm
Key HPs
algorithm_modesubtract_mean
Instances
GPU or CPU — depends on input data specifics.
🔢 Specialized Algorithms
KNN
K-Nearest Neighbors
  • Classification: return most frequent label of K nearest points
  • Regression: return average value of K nearest points
Input
recordIO-protobufCSV
First column = label. Test channel emits accuracy or MSE.
Pipeline
Sample → Dimensionality reduction (sign/fjlt methods) → Build index → Serialize → Query with K
Key HPs
K!sample_size
Instances
Train: CPU or GPU (m5.2xl, p2.xl). Inference: CPU for low latency, GPU for large batch throughput.
Factorization Machines
What it does
  • Handles sparse data: click prediction, item recommendations
  • Supervised: classification or regression
  • Pair-wise interactions only (user → item)
  • Usually for recommender systems
Input
recordIO-protobuf (Float32)
CSV not practical (too sparse).
Key HPs
Initialization methods
Bias, factors, linear terms: uniform, normal, constant.
Instances
CPU recommended. GPU only works with DENSE data.
⚡ Exam: FM = sparse data + recommendations. recordIO-protobuf only. CPU recommended. GPU only for dense data.
IP Insights
What it does
  • Unsupervised learning of IP address usage patterns
  • Identifies suspicious/anomalous IP behavior
  • Detect: anomalous logins, resources created from unusual IPs
Input
CSV only
Format: Entity, IP. No preprocessing needed. Optional validation (computes AUC).
How it works
  • Neural network learns latent vector reps of entities + IPs
  • Entities hashed + embedded (need large hash size)
  • Auto-generates negative samples (random entity-IP pairs)
Key HPs
num_entity_vectorsvector_dimepochs
num_entity_vectors hash size = 2× unique entity IDs. vector_dim too large → overfit.
Instances
GPU recommended (ml.p3.2xlarge+, multi-GPU OK). CPU size depends on vector_dim + num_entity_vectors.
