🔍 Amazon Athena
  • Serverless interactive SQL queries on S3 data
  • Presto under the hood. Data stays in S3.
  • Formats: CSV, TSV, JSON, ORC, Parquet, Avro
  • Compression: Snappy, Zlib, LZO, Gzip
  • Cost: $5 per TB scanned. DDL is free. Failed queries not charged.
  • Save 30–90% cost using columnar formats (ORC, Parquet)
ACID Transactions (Apache Iceberg):
  • Add table_type = 'ICEBERG' to CREATE TABLE
  • Concurrent row-level modifications, time travel queries
  • Use OPTIMIZE table REWRITE DATA USING BIN_PACK for compaction
Performance Optimization:
  • Use columnar data (ORC/Parquet)
  • Fewer large files > many small files
  • Use partitions. After adding: MSCK REPAIR TABLE
Anti-patterns: Athena ≠ formatted reports (use QuickSight), Athena ≠ ETL (use Glue). CTAS trick: use CREATE TABLE AS SELECT to convert CSV→Parquet in S3.
🎛 Feature Engineering — Core Techniques
❓ Handling Missing Data
Mean/Median Replacement
Replace with column mean or median. Fast & easy. Use median for outlier-heavy data. ❌ Misses correlations, can't use on categorical features.
Dropping Rows
Only if few rows affected & won't bias data. Quick but NOT the "best" answer for exam questions. Almost anything is better.
KNN Imputation
Find K nearest rows, average their values. Numerical data only. Can use Hamming distance for categorical.
Deep Learning / Regression
Build a model to impute for your model! Works well for categorical. Advanced: MICE (Multiple Imputation by Chained Equations).
Best answer: Get more real data! KNN or Deep Learning = best imputation. Mean replacement = simplest but least accurate.
⚖️ Handling Unbalanced Data
Large discrepancy between positive/negative cases. e.g., fraud detection — mainly a problem with neural networks.
Oversampling
Duplicate samples from the minority class. Simple but can cause overfitting.
Undersampling
Remove samples from majority class. Discards data — not ideal unless avoiding big data scaling issues.
SMOTE ⭐
Synthetic Minority Over-sampling TEchnique. Generates new samples using KNN (mean of neighbors). Also undersamples majority class. Better than just oversampling.
Adjust Threshold
Increase classification threshold → reduces false positives but increases false negatives. Trade-off decision.
Data Wrangler supports: Random oversampling, undersampling, SMOTE — all built-in!
📊 Outliers
  • Variance (σ²): Avg of squared differences from mean
  • Std Dev (σ): √variance. Data >1σ from mean = unusual
  • "How many sigmas away" = how extreme
  • Remove outliers responsibly — understand WHY
AWS tool: Random Cut Forest detects outliers in QuickSight, Kinesis Analytics, SageMaker
📦 Binning & Encoding
  • Binning: Group values into ranges (20s, 30s, 40s...). Numeric → ordinal. Useful when measurements have uncertainty.
  • Quantile binning: Ensures equal-size bins
  • One-Hot Encoding: Category → binary vector. Column per category, 1 for match, 0 elsewhere. Common in deep learning.
Use when: Model needs categorical data as numbers. Prevents ordinal bias.
📐 Scaling & Transforming
  • Normalization: Scale features to 0–1 or around 0. Required for neural nets. Use MinMaxScaler in scikit-learn.
  • Why scale? Prevents larger-magnitude features from dominating (e.g., age vs. income)
  • Log transform: For exponential-trending data
  • x, x², √x: Allows learning super/sub-linear functions (YouTube recommendation technique)
  • Shuffling: Removes residual order signals from training data
🌀 Dimensionality
  • Curse of Dimensionality: Too many features → sparse data → poor model performance
  • Each feature = a new dimension
  • Dimensionality Reduction:
  • PCA — projects to fewer dimensions while preserving variance
  • K-Means — clusters features together
  • TF-IDF: Term Frequency × Inverse Document Frequency. Measures word importance in a doc relative to all docs. Use log(IDF).
  • N-grams: unigrams, bigrams, trigrams for richer text features

How much outcome distributions of facets diverge

Maximum divergence between outcome distributions
TVD Total Variation Distance
L1-norm difference between outcome distributions
CDD Conditional Demographic Disparity
Disparity of outcomes by subgroups
Explainability Tools:
  • SHAP (Shapley Additive exPlanations): measures impact of dropping each feature. Originated in game theory.
  • Asymmetric Shapley Values: for time series — contribution per time step
  • PDPs (Partial Dependence Plots): shows how feature values influence predictions
Model Monitor + Clarify: Monitors for bias drift + alerts via CloudWatch when new bias emerges in deployed model.
